# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability or concern related to any of the portfolio projects, findings, or methodologies, please report it responsibly.

### How to Report

Please do **not** open a public GitHub issue. Instead:

1. Email security concerns to the repository owner via GitHub
2. Include a detailed description of the vulnerability or concern
3. Provide any supporting evidence or reproduction steps
4. Allow 30 days for response and remediation

### Responsible Disclosure

This portfolio demonstrates security best practices including:
- Professional incident response procedures
- Proper vulnerability disclosure timelines
- Evidence preservation and chain of custody
- Compliance with industry standards (ISO 27001, NIST Cybersecurity Framework)
- Forensic investigation methodologies
- Security architecture principles

## Portfolio Content Notice

These portfolio projects document real-world security scenarios, incident investigations, and security frameworks. All content includes:

- **De-identified incident details** - Names, organizations, and sensitive identifiers have been anonymized
- **Security vulnerability information** - Presented in an educational context
- **Analysis methodologies** - Professional incident response and forensic techniques
- **Security control recommendations** - Based on industry best practices and standards

All sensitive information has been sanitized or anonymized to protect privacy, proprietary information, and security.

## Security Updates

Portfolio projects are maintained with:
- Latest compliance frameworks and standards (ISO 27001, NIST SP 800-53)
- Current incident response procedures based on emerging threats
- Updated threat intelligence and attack methodologies
- Revised security recommendations reflecting industry evolution
- Forensic analysis best practices

## Project-Specific Security Policies

Each portfolio project includes its own `SECURITY.md` file with project-specific security policies and procedures. For detailed security information about a specific project, refer to the `SECURITY.md` file in that project's repository.
